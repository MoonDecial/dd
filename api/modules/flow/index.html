Module: flow

Utilities for controlling the flow and timing of an animation.

  • EveryCallback

all​

▸ all(...tasks): ThreadGenerator

Run all tasks concurrently and wait for all of them to finish.

Example:

// current time: 0s
yield* all(
  rect.fill('#ff0000', 2),
  rect.opacity(1, 1),

chain​

▸ chain(...tasks): ThreadGenerator

Run tasks one after another.

Example:

// current time: 0s
yield* chain(
  rect.fill('#ff0000', 2),
  rect.opacity(1, 1),
);
// current time: 3s

Note that the same animation can be written as:

yield* rect.fill('#ff0000', 2),
yield* rect.opacity(1, 1),

The reason chain exists is to make it easier to pass it to other flow functions. For example:

yield* all(
  rect.corenerRadius(20, 3),
  chain(
    rect.fill('#ff0000', 2),
    rect.opacity(1, 1),
  ),
);

delay​

▸ delay(time, task): ThreadGenerator

Run the given generator or callback after a specific amount of time.

Example:

yield* delay(1, rect.fill('#ff0000', 2));

Note that the same animation can be written as:

yield* waitFor(1),
yield* rect.fill('#ff0000', 2),

The reason delay exists is to make it easier to pass it to other flow functions. For example:

yield* all(
  rect.opacity(1, 3),
  delay(1, rect.fill('#ff0000', 2));
);

yield timer.runner;

loop​

▸ loop(iterations, factory): ThreadGenerator

Run the given generator N times.

Each time iteration waits until the previous one is completed.

Example:

const colors = [
  '#ff6470',
  '#ffc66d',
  '#68abdf',
  '#99c47a',
];

yield* loop(
  colors.length,
  i => rect.fill(colors[i], 2),
});

Parameters​

NameTypeDescription
iterationsnumberNumber of iterations.
factoryLoopCallbackA function creating the generator to run. Because generators can't be reset, a new generator is created each iteration.

Returns​

sequence​

▸ sequence(delay, ...tasks): ThreadGenerator

Start all tasks one after another with a constant delay between.

The function doesn't wait until the previous task in the sequence has finished. Once the delay has passed, the next task will start event if the previous is still running.

Example

yield* sequence(
  0.1,
  ...rects.map(rect => rect.x(100, 1))
);

waitUntil​

▸ waitUntil(event, after?): ThreadGenerator

Wait until the given time event.

Time events are displayed on the timeline and can be edited to adjust the delay. By default, an event happens immediately - without any delay.

Example:

yield waitUntil('event');
